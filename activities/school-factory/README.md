# 🎓 Activity 1: School System Factory

## 🎯 Objective

Apply the **Factory Pattern** to improve system design and manage object creation effectively.

---

## 📌 Scenario

You are building a **School System**.

The system includes multiple classes:

- School
- Teacher
- SchoolEvent

Each part of the system needs to create **different types of students**.

---

## ⚠️ Problem Context

Object creation is currently handled in multiple places across the system.

As the system grows, this approach may lead to:
- Repeated logic
- Difficult maintenance
- Inconsistent behavior

---

## 🧠 Your Task

- Analyze how student objects are created in the system  
- Redesign the system using the **Factory Pattern**  
- Ensure object creation is handled in a clear and maintainable way  

---

## 📏 Requirements

- Define a common abstraction for students  
- Support multiple student types  
- Centralize object creation  
- Keep the design clean and scalable  

---

## ✅ Expected Outcome

- A well-structured design using a factory  
- Clear separation of responsibilities  
- Easy extension for new student types  

---

## 🧩 Challenge Extension (Optional)

- How would your design handle frequent addition of new student types?  
- Can you improve flexibility without making the system more complex?
